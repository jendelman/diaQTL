diaQTL Vignette: Potato Example

Structure of the input files

The example data set is a 3x3 half-diallel population in potato. In other words, three founders were crossed in all three pairwise combinations. Three input files are needed to run the QTL analysis: (1) pedigree file, (2) genotype file, (3) phenotype file.

1) Pedigree file

The pedigree file has four columns: id, population, mother, and father. In the linear model, the mean of each population is estimated as a fixed effect. This allows control over whether reciprocal crosses are assumed to have the same mean (label as one population) or different means (label as separate populations).

pedcsv <- system.file("vignette_data", "potato_ped.csv", package = "diaQTL")
ped <- read.csv(pedcsv, as.is = T)

2) Genotype file

The first three columns of the genotype file contain the marker name, chromosome, and position in either cM or bp. Subsequent columns contain the genotype probabilities for each individual. The position column in the potato dataset is cM.

genocsv <- system.file( "vignette_data", "potato_geno.csv", package = "diaQTL" )
geno <- read.csv( genocsv, as.is = T, check.names = F )
geno[1:5,1:3]
##                marker chromosome position
## 1 solcap_snp_c2_36608          1     0.00
## 2 solcap_snp_c2_36658          1     0.22
## 3 solcap_snp_c1_10930          1     0.41
## 4       PotVar0120130          1     0.58
## 5       PotVar0120070          1     0.97

Genotype probabilities are encoded as strings, following the format exported by the PolyOrigin software:

geno[1,10]
## [1] "22|31|32|35|52|62=>0.083|0.004|0.883|0.004|0.002|0.022"

The integers separated by | on the left side of the equal sign refer to genotype states, and the decimal numbers on the right side of the equal sign are probabilities. Only nonzero probabilities need to be included. There are 100 possible states for F1 populations, and 35 possible states for S1 populations:

Each state has four integers, separated by dashes, to indicate which parental chromosomes were inherited. For F1 populations, the maternal chromosomes are labeled 1-4 and the paternal chromosomes 5-8.

3) Phenotype file

In the phenotype input file, the first column should be the individual identifier, followed by columns for different traits, and then optionally any columns with fixed effects to include in the linear model (e.g., block, environment). Only one trait, tuber shape, is provided in the example potato data set.

phenocsv <- system.file( "vignette_data", "potato_pheno.csv", package = "diaQTL" )
pheno <- read.csv( phenocsv, as.is = T )
head( pheno )
##            id tuber_shape
## 1  W15277B-1R       3.726
## 2 W15268-146R       3.013
## 3 W15277B-19R       2.998
## 4  W15263-44R       2.965
## 5  W15268-30R       2.720
## 6   W15267-6R       2.684
hist(pheno$tuber_shape,main="",xlab="Tuber shape")

Tuber shape in this data set is defined as -log(L/W - 1), where L/W is the average length/width ratio of tubers weighing 6-10 ounces (170-285g). Higher values correspond to rounder tubers.

After installing and attaching the package, use read_data to read in all three files. If there are fixed effects in the phenotype input file, they need to be specified as well (see reference manual).

Setting parameters

diaQTL uses Bayesian regression methods from R package BGLR, which are based on Markov Chain Monte Carlo (MCMC). The function set_params is used to determine the total number of iterations (nIter) and the number that should be discarded as burn-in. Consult the reference manual on how to change the tolerance parameters if needed. Because MCMC is a stochastic process, the results will not be the same each time.

params <- set_params( diallel_example, trait = "tuber_shape" )
params
## $burnIn
## [1] 18
## 
## $nIter
## [1] 496

Single QTL scan

The scan1 function performs a single QTL scan by regression of the phenotypes on the genotype probabilities. By default, dominance is not incuded in the scan because it takes longer. However, there may be traits where dominance is needed for QTL detection, or dominance could lead to refinement of the QTL position. It is also possible to restrict the QTL scan to a subset of chromosomes, or to use multiple cores for faster execution on Linux or UNIX (including MacOS) systems. See the reference manual for these options.

scan1_example <- scan1( data = diallel_example,
                      trait = "tuber_shape",
                      params = params )

Detection threshold

The function LODthresh can be used to estimate the LOD threshold corresponding to a genome-wide false positive rate of 0.05. This function uses a curve fit to simulated datasets with no QTL. The genome size for the potato dataset can be estimated from the input map.

potato.genome <- sum(tapply(geno$position,geno$chromosome,max))/100 #Morgans 
threshold <- LODthresh(genome.size=potato.genome,num.parents=3,ploidy=4)
cat(threshold)  #print threshold to standard output
## 6.7

Another option is to perform a permutation test with scan1_permute, which is computationally intensive.

scan1_permute_example <- scan1_permute( data = diallel_example, 
                                   trait = "tuber_shape", 
                                   n.permute = 1000, 
                                   params = params, 
                                   n.core = 4 )

The result from scan1_permute is a data frame with the LOD score for each permutation, which can be used to determine the threshold for \(\alpha = 0.05\):

alpha <- 0.05
quantile( scan1_permute_example$LOD, probs = 1-alpha )
##      95% 
## 6.543234

The result is very similar to the threshold from the LODthresh function.

The results show there is a QTL on chromosome 10 that explains \(R^{2} = 38\%\) of the variance. The gene underlying this QTL is most likely StOVP20 (Wu et al. 2018; doi.org/10.1038/s41467-018-07216-8). The most significant marker is solcap_snp_c2_255222, which can be passed to the fitQTL function to get the estimated parental allele effects, as both a table (effectsA) and barplot (plotA). Use the dominance=T option to include dominance effects, which are returned as effectsD (table) and plotD (heatmap). To improve the precision of the results, the number of MCMC iterations is increased compared to that used for scan1.

fit1 <- fitQTL( data = diallel_example, 
                       trait = "tuber_shape", 
                       params = list(burnIn=100,nIter=3000), 
                       marker = "solcap_snp_c2_25522")

#With dominance
fit2 <- fitQTL( data = diallel_example, 
                       trait = "tuber_shape", 
                       params = list(burnIn=100,nIter=3000), 
                       marker = "solcap_snp_c2_25522",
                       dominance = T)

#Compare DIC for additive vs. dominance model
print(data.frame(Add=round(fit1$deltaDIC,0),Dom=round(fit2$deltaDIC,0)),row.names=F)
##   Add  Dom
##  -176 -196

Because the dominance model has more parameters, a penalized measure of goodness-of-fit is needed to determine whether it is superior to the additive model. fitQTL returns the Deviance Information Criterion (DIC) for the QTL model relative to the null model (no QTL) as deltaDIC. Lower DIC values are better, so the dominance model is selected in this case. The amounts of additive and dominance variance relative to the total variance are returned as \(h^2\) and \(d^2\), respectively. In this case, the dominance variance is about half the additive variance.

data.frame(h2=round(fit2$h,2),d2=round(fit2$d2,2))
##            h2   d2
## Estimate 0.31 0.14
## SE       0.01 0.00
fit2$plotA

fit2$plotD

The largest effect allele is present on homolog 4 of W6511-1R. The function dosage can be used to extract the allele dosage of the parental alleles at this locus.

roundQTL <- dosage( data = diallel_example, 
                    marker = "solcap_snp_c2_25522")
hist(roundQTL[,"W6511-1R.4"],main="",xlab="Allele Dosage")

which(roundQTL[,"W6511-1R.4"] > 1.5)
## W15268-53R W15268-93R W15267-10R 
##         37         59        188

The result shows there are three individuals with two copies of the W6511-1R.4 allele, which is possible due to double reduction. This occurs when a quadrivalent forms in meiosis I and sister chromatid fragments migrate to the same pole in meiosis II. The function plot_dosage can be used to visualize the pattern of recombination between parental homologs.

plot_dosage( data = diallel_example, 
             id = "W15268-53R", 
             chrom = 10,
             distance = "cM",
             marker = "solcap_snp_c2_25522" )

The dark blue segment indicates two copies of the W6511-1R.4 allele, and the dashed vertical line shows the position of the round QTL.

Multiple QTL mapping

In the absence of epistasis, multiple QTL on different chromosomes can be detected adequately with scan1. If there is epistasis, or if the QTL are on the same chromosome, other methods are needed. At present, the only option available in diaQTL is to conduct a second analysis with scan1 using the most significant marker from the first scan as a cofactor. Consult the reference manual for more details.
